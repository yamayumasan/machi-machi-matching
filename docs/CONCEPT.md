# コンセプト

## サービス概要

**マチマチマッチング**は、「やりたいこと」を双方向で表明し、自動でマッチングされる地域密着型活動マッチングサービスです。

### 名前の由来

- **マチ（街）**: 地域密着型。特定エリアでの活動マッチング
- **マチ（待ち）**: 「やりたいこと」を表明しておくだけでオファーが届く
- **マッチング**: 募集者と参加希望者を自動でつなぐ

---

## 解決する課題

### ユーザーが抱える問題

1. **やりたいことはあるが、募集を探すのが面倒**
   - 大量の投稿から自分に合うものを探す必要がある
   
2. **自分で募集を立てるほどではない**
   - 誘われたら参加したいが、主催するのはハードルが高い
   
3. **思案段階のアイデアを発信しにくい**
   - 確定したイベントしか投稿できない雰囲気

4. **既存サービスのUIがわかりにくい**
   - ジモティー、つなげーと等は操作が複雑

### 既存サービスの限界

| サービス | 問題点 |
|---------|--------|
| **ジモティー** | 取引中心、コミュニティ感なし、UI古い |
| **つなげーと** | 料金体系複雑、本名晒される、ナンパ目的混在 |
| **Meetup** | 英語中心、日本でのユーザー少ない |
| **恋活アプリ** | 友達作りは副次的、異性限定 |

## ソリューション

### コアコンセプト：双方向の意思表示

| 役割 | 行動 | 例 |
|------|------|-----|
| **募集者** | 募集を作成 | 「来週末ボドゲやりたい人いませんか？」 |
| **参加希望者** | 「今やりたいこと」を表明 | 「ボドゲやりたい気分🎲」 |

**両者の「やりたいこと」がマッチした時に、相互に提案される**

### 差別化ポイント

| 観点 | 既存サービス | マチマチマッチング |
|------|------------|------------------|
| 投稿のハードル | 高い（詳細必須） | **低い（ゆるくてOK）** |
| 募集者→参加者 | 一方向 | **双方向（オファー可）** |
| マッチング | 手動検索 | **自動提案** |
| 参加者の行動 | 募集を探す | **表明しておくだけ** |
| グループ形成 | 参加後 | **1人から即グループ** |

## ターゲットユーザー

### メインターゲット

- **20〜40代の社会人**
- 趣味の仲間を探したいが、恋活アプリには抵抗がある
- 自分で募集を立てるほどではないが、誘われたら参加したい
- 新しい街に引っ越してきて友達がいない

### ユースケース

1. **趣味仲間探し**
   - ボードゲーム、カフェ巡り、スポーツ、etc.

2. **ゆるい飲み会**
   - 「今週末暇な人、飲みませんか？」

3. **イベント同行**
   - 「〇〇のライブ、一緒に行く人いませんか？」

4. **新しい活動の仲間募集**
   - 「プログラミング勉強会やりたいんだけど…」

## 市場機会

### 日本の友達作り市場の課題

- 恋活・婚活アプリ市場は517億円以上、成長継続
- しかし「友達作り専用」のアプリは未発達
- 30代以上の「新しい友達を作りにくい」層へのニーズ大

### 競合が弱い理由

1. **ジモティー**: 取引・売買が中心で、活動マッチングは副次的
2. **つなげーと**: 運営体制・料金体系に問題（レビュー評価低い）
3. **海外サービス**: Meetup、Bumble BFFは日本語対応弱い

## 成功指標（KPI）

### MVP段階

- 登録ユーザー数: 500人（東京・仙台合計）
- 月間アクティブ募集数: 100件
- マッチング成立率: 30%以上
- グループ形成後の実施率: 50%以上

### グロース段階

- MAU: 10,000人
- 月間マッチング数: 1,000件
- 有料転換率: 5%
